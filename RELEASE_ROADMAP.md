# 🚀 앱 스토어 출시 로드맵 (BURN IT)

앱을 스토어(Google Play Store, Apple App Store)에 정식 출시하기 위한 상세 준비 가이드입니다. 본격적인 출시는 **행정적 준비 -> 디자인/리소스 준비 -> 기술적 준비 -> 빌드 및 배포** 순서로 진행됩니다.

---

## 📅 Step 1: 행정적 준비 (계정 및 비용)

출시를 위해서는 각 스토어의 개발자 계정이 필요합니다. 승인까지 시간이 걸릴 수 있으므로 가장 먼저 미리 생성하는 것을 권장합니다.

### 1. Google Play Console (Android)
*   **비용:** $25 (약 3만 4천원, **1회성** 결제)
*   **준비물:** 해외 결제 가능한 신용카드, 구글 계정
*   **절차:** [Google Play Console](https://play.google.com/console) 접속 -> 계정 생성 -> 결제 -> 신원 인증 (여권/운전면허증 필요)

### 2. Apple Developer Program (iOS) (선택 사항)
*   **주의:** Windows 환경에서는 직접 iOS 빌드가 불가능합니다. (Mac 필요)
*   **비용:** $99/년 (약 14만원, 매년 갱신)
*   **준비물:** Apple ID (2단계 인증 필수), D-U-N-S 번호 (법인인 경우)
*   **참고:** Windows만 사용 중이라면, Android 출시를 우선 진행하거나 클라우드 빌드 서비스(Codemagic 등)를 이용해야 합니다.

---

## 🎨 Step 2: 디자인 및 리소스 준비

스토어 등록 정보에 들어갈 시각 자료와 텍스트입니다. 사용자에게 앱을 어필하는 가장 중요한 요소입니다.

### 1. 앱 아이콘 (App Icon)
*   **규격:** 1024 x 1024 px (PNG, 투명 배경 없음)
*   **설명:** 홈 화면에 설치될 고해상도 아이콘입니다. '불', '감정', '쓰레기통' 등의 키워드를 잘 표현하는 깔끔한 디자인이 필요합니다.

### 2. 스토어 그래픽 (Google Play)
*   **그래픽 이미지 (Feature Graphic):** 1024 x 500 px (PNG/JPG)
    *   스토어 상단에 노출되는 배너 이미지입니다. 앱의 분위기를 잘 나타내는 아트워크가 필요합니다.
*   **휴대폰 스크린샷:** 최소 2장 ~ 최대 8장 (16:9 또는 9:16 비율 권장)
    *   주요 기능(홈 화면, 감정 태우기, 광장, 보관함)을 캡처하여 설명 텍스트와 함께 디자인하면 좋습니다.

### 3. 앱 소개 텍스트
*   **앱 이름:** BURN IT (또는 한국어 제목: 번잇 - 감정 쓰레기통) (30자 이내)
*   **간단한 설명:** 사용자를 훅(Hook)할 수 있는 한 줄 소개 (80자 이내)
    *   예: "마음속 분노, 참지 말고 태워버리세요. 익명 감정 해소 커뮤니티."
*   **자세한 설명:** 기능 소개, 사용 방법, 기대 효과 등을 상세히 서술 (4000자 이내)

## ⚖️ Step 3: 법적 문서 준비 (필수)

스토어 등록 시 반드시 필요한 법적 고지 문서입니다.

### 1. 개인정보 처리방침 (Privacy Policy)
*   **내용:** 어떤 정보를 수집하는지(앱은 닉네임, 감정 데이터 등을 로컬에 저장), 제3자 제공 여부(없음) 등을 명시해야 합니다.
*   **방법:** 무료 생성 사이트(예: [개인정보보호 포털](https://www.privacy.go.kr)) 이용 후, 내용을 **Notion 페이지**나 **블로그** 등에 게시하고 해당 URL을 준비하세요.

## ☁️ Step 4: 백엔드 구축 (Firebase 연동) - **전략 B 적용**

실제 유저들이 서로의 글을 보고 소통(장작/물뿌리기)할 수 있도록 **Firebase**를 도입합니다.

### 1. Firebase 프로젝트 생성
*   [Firebase Console](https://console.firebase.google.com/)에서 새 프로젝트를 생성합니다.
*   **Authentication:** '익명 로그인'을 활성화합니다. (유저는 별도 가입 절차 없이 바로 이용 가능, 기기 변경 시 연동을 위해 '구글 로그인'도 추후 고려 가능)
*   **Cloud Firestore:** 실시간 데이터베이스를 생성합니다. 광장 게시글(`posts`) 컬렉션을 설계합니다.

### 2. 앱에 Firebase SDK 연결
*   Flutter 프로젝트에 `firebase_core`, `cloud_firestore`, `firebase_auth` 패키지를 추가합니다.
*   `flutterfire configure` CLI 명령어를 통해 앱과 Firebase 프로젝트를 연결합니다.

### 3. 기능 개발 (추가 구현 필요 사항)
*   **실시간 연동:** `VentingViewModel`의 광장 로직을 로컬 메모리 리스트에서 **Firestore 스트림**으로 변경해야 합니다.
*   **상호작용 처리:** 장작/물뿌리기 클릭 시 DB 트랜잭션을 통해 카운트를 실시간으로 증가시킵니다.
*   **신고 기능(필수):** 커뮤니티 앱 출시 시 스토어 필수 요구사항인 '사용자 차단' 및 '부적절한 콘텐츠 신고' 기능을 구현해야 합니다.

## 🛠️ Step 5: 기술적 준비 (코드 작업)

출시용 버전을 만들기 위해 앱 내 설정을 변경해야 합니다.

### 1. 앱 아이콘 적용
*   `flutter_launcher_icons` 패키지를 사용하여 고해상도 아이콘을 앱 프로젝트에 적용합니다.

### 2. 패키지 이름(Package Name) 확인
*   현재: `com.example.emotiontok`
*   **변경 필요:** `com.example`은 스토어 업로드가 불가능할 수 있습니다. 고유한 도메인 형식으로 변경해야 합니다. (예: `com.monitoro.burnit`)
    *   *이 작업은 Firebase 연동 시 패키지 이름이 중요하므로, Firebase 설정 **전에** 먼저 수행해야 합니다.*

### 3. 서명 키(Keystore) 생성 (Android)
*   앱의 신분증과 같은 파일입니다. **절대 분실하면 안 되며**, 분실 시 앱 업데이트가 영구적으로 불가능해집니다.
*   Windows 터미널에서 생성 명령어를 실행해야 합니다.

### 4. 권한 및 설정 최종 점검
*   `AndroidManifest.xml`: 불필요한 권한 삭제.
*   앱 이름 표시 설정 (`android:label`).

## 🚀 Step 6: 빌드 및 배포

모든 준비가 끝나면 실제 파일을 생성하여 업로드합니다.

### 1. Android App Bundle (.aab) 빌드
*   APK보다 효율적인 배포 포맷인 AAB 파일로 빌드합니다.
*   명령어: `flutter build appbundle --release`

### 2. 스토어 업로드 및 심사
*   Google Play Console에 접속하여 새 앱을 생성하고, 정보를 입력한 뒤 AAB 파일을 업로드합니다.
*   **내부 테스트** 트랙을 활용하여 실시간 DB 작동 여부를 먼저 확인해야 합니다.
